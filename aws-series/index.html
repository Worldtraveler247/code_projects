How to use this series

Every service page has two parts: an overview (what / who / why) and a tutorial (how to actually run it). The tutorials are written in three tracks side-by-side:

🖱️ Console

Click-by-click in the AWS Console. Best for first time and for visualizing what the abstractions look like.

⌨️ AWS CLI v2

Same operation as a CLI command. Best for scripting, repetition, and learning the underlying API.

🏗️ Terraform

Same operation as infrastructure-as-code. Best for production, code review, and reproducing across environments.

Security Mindset

Every tutorial uses named AWS profiles, IAM roles over long-lived keys, and least-privilege policies. We'll call out blast radius and cleanup steps every time — so you don't end up with a rogue NAT Gateway costing you $32/month for a lab you forgot.

Prerequisites

  • An AWS account (free tier covers most of these labs).
  • AWS CLI v2 installed and configured with a named profile.
  • Terraform 1.7+ for the IaC track.
  • Comfort with a terminal (bash/zsh) — RHCSA-level Linux is more than enough.
